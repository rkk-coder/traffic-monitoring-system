Traffic Monitoring System

Система бэкенда для мониторинга и анализа дорожного движения в реальном времени. Позволяет отслеживать транспортные средства, фиксировать их координаты и скорость, а также автоматически обнаруживать опасные ситуации на дороге.

Описание

Traffic Monitoring System — это REST API для сбора и анализа данных о транспортных средствах. Система принимает данные о машинах (координаты, скорость), сохраняет их в базу данных и предоставляет аналитику, включая обнаружение слишком близко расположенных автомобилей (менее 5 единиц), расчёт средней скорости потока и общую статистику по автомобилям. Проект демонстрирует навыки работы с FastAPI, SQLAlchemy ORM, SQLite и базовой геометрической аналитикой.

Возможности

Добавление автомобилей — сохранение данных о машине (ID, координаты X/Y, скорость). Обнаружение опасных ситуаций — автоматическое предупреждение при сближении автомобилей на дистанцию менее 5 единиц. Аналитика трафика — расчёт средней скорости и общего количества машин. Хранение данных — SQLite через SQLAlchemy ORM. Интерактивная документация API — автоматически генерируется Swagger UI.

Технологии

FastAPI, SQLAlchemy 2.0+, SQLite, Pydantic, Python (math).

Установка и запуск

Требования: Python 3.8 или выше. Шаги: клонировать репозиторий, создать виртуальное окружение, установить зависимости командой pip install -r requirements.txt, запустить сервер командой uvicorn main:app --reload. После запуска открыть в браузере http://localhost:8000, документация Swagger доступна по адресу http://localhost:8000/docs.

API Endpoints

POST /cars/ — добавить автомобиль. Тело запроса: {"id": 1, "x": 10.5, "y": 20.3, "speed": 45.2}. Ответ: {"message": "Car added"}. GET /analyze/ — получить анализ трафика. Ответ: {"warnings": ["Cars 1 and 2 too close"], "average_speed": 42.5, "cars_count": 10}.

Пример использования

import requests
requests.post("http://localhost:8000/cars/", json={"id": 1, "x": 10.0, "y": 20.0, "speed": 60.0})
analysis = requests.get("http://localhost:8000/analyze/")
print(analysis.json())

Структура проекта

main.py — точка входа FastAPI, маршруты; models.py — SQLAlchemy модели (Car); schemas.py — Pydantic схемы для валидации; database.py — подключение к БД и сессии; analytics.py — логика анализа (расстояния, скорость); requirements.txt — зависимости; README.md — этот файл.
